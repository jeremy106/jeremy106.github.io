Javascript Fundamentals

Javascript vs HTML vs CSS

A modern webpage is (usually) written using three separate programing languages/frameworks. HTML is used to define the content of the page, CSS is used to define the layout and visual appearance, but it's Javascript that is the secret sauce which transforms a webpage from a static document to the kind of dynamic and interactive digital media we're accustomed to. If loading a website was like putting together a model aeroplane, HTML would be the list of pieces you get in the box, CSS would be the instructions on how all the pieces fit together to make it look like a plane, and Javascript would be a bunch of electronics you could install to make it go NYEEEOOM when you fly it around your room

Much like our model plane, the sky is the limit with Javascript in terms of what you can build given enough effort. However you can achieve a lot just by mastering some of the fundamentals outlined below

Control Flow

Control flow refers to the order in which a computer progam carries out all of the instructions it has been given. Most of the time a program will run one line after another in the order in which they're written. But what if we want to make our program do something different depending on what has happened beforehand? To make a useful program, we need a way for our code to make decisions about what to do. One option is to use an if/else statement.

// How to respond when the empty light in your car comes on
if ( petrolLightIsOn ) {
  return pretendNotToNotice()
  console.log("I hope my wife goes to the petrol station soon...")
} else {
  return keepOnDriving()
}

An if/else statement works a bit like it sounds. If some condition is met (e.g. the petrol light is on) then execute a particular command (e.g. just kinda ignore it...). Otherwise, execute some other command. With enough ifs and elses you can break down pretty much any complex decision

Loops

One of the best things about computers is that they never complain about doing the same thing over and over again. Repeating a simple operation over and over again is a common occurence in computer programming and it is usually achieved with loops

for ( let i=1, i<dirtyDishes.length, i++) {
 let dish = dirtyDishes[i]
 let cleanDish = scrub(dish)
 dishRack.push(cleanDish)
}

The loop above is doing something we all wish we could ask a robot to do: Clean the dishes. The syntax of a loop can be confusing at first glance, but all its really saying is:

  • Start counting here:  1
  • Keep counting until you get here:  The number of things in the dirtyDishes array
  • Every time you finish the task below update the count:  Increase by 1
So in real world terms, Starting with the first dirty dish, it takes the dish from the dirty pile (an array), scrubs the dish, then puts it in the clean pile (another array). It reapeats this for one dish one at a time until all the dirty dishes are clean.

Arrays vs Objects

Arrays and Objects are both used to store information that our program might need, but do so in different ways. An Array is a bit like a bucket. It is a generic container that you can put pretty much anything into, either for storage or to keep things organised. Unlike a bucket however, everyhing that goes into an Array is assigned a position (index) so that we can easily retrieve items from it.

//An array containing some of my favourite extraterrestial satellites
 let moons = ['Phobos', 'Deimos', 'Europa', 'Ganymede']

An Object is also a container for information, but each entry exists as a combination if a name and a value. Its a bit like an ID card which lists various attributes or features about the thing it represents

//An object which stores some interesting moon properites
let deimos = {
 orbitsPlanet: "Mars",
 radiusKm: 6.27 ,
 craters: ["Voltaire","Swift"],
}
